# Template — User Story

> Estrutura padrão, sem conteúdo de domínio. Preencher os campos entre `< >` ao gerar uma história real.

## Formato narrativo

```
Como <ator/persona>,
Quero <ação/capacidade>,
Para que <benefício/valor>.
```

## Campos complementares

- **ID**: `<identificador único, ex.: US-001>`
- **Título**: `<resumo curto da história>`
- **Epic/Feature relacionada**: `<referência ao Epic ou Feature pai, se houver>`
- **Ator(es)**: `<persona(s) ou perfil(is) de usuário envolvido(s)>`
- **Descrição**: `<narrativa Como/Quero/Para que acima>`
- **Critérios de Aceitação**: `<lista de critérios — ver template acceptance_criteria.md>`
- **Regras de Negócio**: `<lista de regras aplicáveis — ver template business_rule.md>`
- **Prioridade**: `<MoSCoW, número ou ordenação no Product Backlog>`
- **Estimativa**: `<story points ou estimativa equivalente>`
- **Dependências**: `<outras histórias, sistemas ou decisões das quais esta depende>`
- **Observações**: `<notas adicionais, riscos, suposições>`

## Checklist de qualidade

Antes de considerar a história pronta, verificar contra o critério INVEST (ver `../methodology/invest.md`) e, quando aplicável, contra a Definition of Ready do time (ver `../methodology/dor.md`).
